# Public Release Checklist

Complete this checklist before making the repository public.

## Authorship and Permission

- [ ] All project members have reviewed the public description.
- [ ] The supervisor has approved the public release, if required.
- [ ] Individual contributions are described accurately.
- [ ] The repository does not imply sole authorship by the maintainer.

## Privacy

- [ ] Student IDs have been removed.
- [ ] Personal signatures have been removed.
- [ ] Private email addresses and phone numbers have been removed.
- [ ] The original report PDF has been redacted or withheld.

## Code and Reproducibility

- [ ] Code runs in a clean environment.
- [ ] Dataset paths are configurable.
- [ ] Random seeds and split logic are included.
- [ ] Training commands are documented.
- [ ] Evaluation metrics reproduce the reported tables.
- [ ] No passwords, API keys, tokens, or server credentials are present.

## Licensing

- [ ] Third-party code licenses have been checked.
- [ ] Baseline implementations are credited.
- [ ] A repository license has been selected only after ownership is confirmed.

## Presentation

- [ ] `YOUR_GITHUB_USERNAME` has been replaced everywhere.
- [ ] Broken links and images have been checked.
- [ ] Repository description and topics have been added.
- [ ] The HSI repository is pinned on the profile.
